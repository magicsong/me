# 增强的TODO管理系统 - 使用指南

## 功能概述

基于三层标签分类系统的增强TODO管理，帮助你更智能地优先级排序和精力分配。

### 三个必填标签分类

#### 一、决策类标签（最重要）- 帮你判断"先做什么"

判断任务的紧急程度和收益，核心是优先级决策。

| Emoji | 标签名 | 说明 |
|------|------|------|
| 🔥 | 高风险 | 风险大，必须立即处理，影响线上、关键业务或SLA |
| 🎯 | 高收益 | 投入产出比高，能带来显著业务价值或技术提升 |
| ⛔ | 阻塞中 | 被其他任务或外部依赖阻塞，需要等待 |
| 🧨 | 技术债 | 需要偿还的技术债，改进系统质量 |
| 🧪 | 实验性 | 探索性工作，风险未知，需要验证 |

**何时选择：**
- 🔥 优先度最高（P0）
- 🎯 有明确目标和预期收益（P1）
- 🧨 持续改进（中优先级）
- 🧪 创新探索（视情况）
- ⛔ 标记为阻塞（不影响今天的计划）

---

#### 二、领域类标签（按实际工作）- 避免碎片化

为任务分类，能看出精力投入分布，识别是否过度碎片化。

| Emoji | 标签名 | 说明 |
|------|------|------|
| ☸️ | Kubernetes | K8s相关 |
| 🤖 | Agent / LLM | AI/LLM相关 |
| 🛡 | 稳定性 | 可用性、容错、恢复 |
| 🔐 | 安全 | 安全防护、加密、权限 |
| 📊 | 数据 / 观测 | 数据、监控、日志 |
| 🧩 | 架构设计 | 系统设计、重构 |

**何时选择：**
根据任务涉及的主要领域选择，每个任务可以关联多个领域，但应有一个主要领域。

**精力分配检查：**
- 周报告时看各领域占比
- 发现碎片化问题（太多小任务）
- 识别投入不足的领域

---

#### 三、工作性质标签 - 区分"产出型"和"救火型"

分辨工作的本质，帮助维持产出和稳定的平衡。

| Emoji | 标签名 | 说明 |
|------|------|------|
| 🧱 | 基建 | 基础设施建设、工具链 |
| 🚀 | 推进中项目 | 正在进行的主要项目 |
| 📝 | 文档 / 输出 | 文档、文章、总结 |
| 🔍 | 调研 | 技术调研、方案论证 |
| 🧯 | 故障 / Oncall | 应急响应、生产问题 |

**何时选择：**
- 🧯 故障和oncall（应该最小化）
- 🚀 核心项目推进（应该最大化）
- 📝 知识输出（必要的投入）
- 🧱 基建和工具（定期投入）
- 🔍 调研（计划阶段）

**工作平衡检查：**
- 每周应该有60%以上的时间在🚀项目上
- 🧯 故障处理应控制在15%以内
- 其他工作20-25%

## 使用场景

### 场景1：创建新TODO

1. 点击"快速添加待办事项"或"创建任务"
2. 输入任务标题和描述
3. 选择优先级
4. **必须选择三个必填标签分类各一个**
   - 点击"选择标签"打开分类选择器
   - 每个分类选择最适合的标签
5. 可选：添加其他标签
6. 提交

### 场景2：编辑任务标签

1. 在任务卡片上点击"标签"按钮
2. 点击"必填标签"区域的"选择标签"
3. 重新选择三个分类的标签
4. 也可以添加其他可选标签
5. 提交

### 场景3：查看精力分布

1. 进入TODO管理页面
2. 按领域类标签筛选
3. 观察各领域任务数量和优先级
4. 调整计划，优化精力分配

### 场景4：缩小任务范围

1. 用🔥高风险标记真正的紧急事项
2. 用🧨技术债标记改进性工作
3. 用🎯高收益标记重点项目
4. 定期清理，避免标签通胀

## 标签管理

### 访问标签管理

路径：`/todolist/tags` 或从TODO管理页面进入

### 功能

- **按分类查看**：切换标签分类选项卡
- **添加标签**：
  1. 输入标签名称
  2. 选择分类
  3. 选择颜色
  4. 点击添加
- **编辑标签**：点击标签名上的编辑图标
- **删除标签**：点击标签名上的删除图标

### 最佳实践

- 不要创建超过20个标签（会导致选择困难）
- 标签名称简洁（2-5个字）
- 预定义的标签充分满足需求，谨慎添加

## 工作流建议

### 早上规划

1. 打开TODO管理
2. 查看未完成的高优先级任务
3. 按🔥、🎯、🚀 筛选今天要做的
4. 关注标签分布，确保不过度碎片化

### 过程中

1. 完成任务时标记完成
2. 如果发现阻塞，更新为⛔标签
3. 新任务随时添加并正确分类

### 周总结

1. 查看本周各领域工作占比
2. 分析产出型vs救火型比例
3. 识别下周需要调整的方向
4. 检查是否有过多的🧨技术债

## API相关

### 更新标签时需要的字段

```json
{
  "id": 1,
  "name": "高风险",
  "color": "#EF4444",
  "kind": "todo",
  "category": "decision_type"
}
```

### 三个标签分类值

- `decision_type` - 决策类
- `domain_type` - 领域类
- `work_nature` - 工作性质

## 故障排除

### 问题：无法提交TODO，提示"请先选择所有必填标签分类"

**原因**：没有选择三个分类各一个标签

**解决**：
1. 点击"选择标签"按钮
2. 从决策类选一个（如🎯高收益）
3. 从领域类选一个（如🚀Kubernetes）
4. 从工作性质选一个（如🚀推进中项目）
5. 确认后提交

### 问题：标签选择对话框不显示某些标签

**原因**：标签没有分类，或分类不正确

**解决**：进入标签管理，编辑相关标签的分类信息

### 问题：想更改已选标签

**解决**：编辑任务 → 点击标签 → 在标签对话框中更改
